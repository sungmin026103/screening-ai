SR Safe-Exclude Triage Platform — 실행 및 사용법
================================================

버전 1.0.2: 첫 라벨링 파일을 1차_제목_라벨링용.xlsx로 단순화했습니다.
첫 시트는 순번, 연도, 제목, 판정(o/x) 열만 제공합니다.

[가장 간단한 실행]
1. 압축을 풉니다.
2. START_SR_PLATFORM.bat를 더블클릭합니다.
3. 최초 실행 때는 전용 환경을 자동 설치하므로 몇 분 걸릴 수 있습니다.
4. 다음부터는 같은 파일을 더블클릭하면 바로 실행됩니다.

[설치와 실행을 따로 하고 싶을 때]
1. 01_FIRST_INSTALL.bat를 최초 1회 실행합니다.
2. 이후에는 02_RUN_PLATFORM.bat를 실행합니다.

[브라우저 사용]
1. 실행 파일을 더블클릭합니다.
2. 브라우저가 자동으로 열립니다.
3. 주소는 일반적으로 http://localhost:8501 입니다.
4. 사용 중에는 검은 실행창을 닫지 않습니다.

[같은 연구실 네트워크에서 주소로 같이 접속]
1. 플랫폼을 실행할 메인 PC에서 03_RUN_LAB_NETWORK.bat를 실행합니다.
2. 창에 표시되는 메인 PC의 IPv4 주소를 확인합니다.
3. 다른 연구실 PC에서 http://메인PC_IP:8501 로 접속합니다.
4. 이 모드는 로그인 기능이 없습니다. 같은 네트워크에서 주소를 아는 사람은 프로젝트와
   결과에 접근할 수 있으므로 신뢰할 수 있는 연구실 내부망에서만 사용하세요.
5. 최초 실행 때 Windows 방화벽 허용 창이 뜰 수 있습니다.

[분석 흐름]
1. 프로젝트명, PICO, 포함/배제기준을 입력합니다.
2. Excel/CSV/RIS 제목 파일을 올리고 제목·초록·연도·인간 판정 열을 연결합니다.
3. 인간 Include/Exclude 라벨이 각 10편 이상이면 모델 학습, OOF 성능평가,
   전체 예측, Excel, 피규어, 감사표본이 한 번에 생성됩니다.
4. 인간 라벨이 부족하면 1차_제목_라벨링용.xlsx가 먼저 생성됩니다.
   판정(o/x) 열에 o/x를 입력한 뒤, 다음 실행에서 이전 라벨 파일로 올립니다.
5. 반복할수록 새로운 인간 판정이 누적되어 프로젝트별 모델이 갱신됩니다.

[결과 해석]
- INCLUDE_REVIEW_FIRST: 먼저 검토할 문헌
- MANUAL_REVIEW: 임계값 사이의 수동 검토 문헌
- LOW_PRIORITY_EXCLUDE_AUDIT: 정말 낮은 우선순위의 배제 후보
- AI_Prediction의 Exclude는 최종 배제가 아닙니다.
- 04_LOW_PRIORITY_AUDIT를 사람이 점검한 뒤에만 실제 자동배제 정책을 결정하세요.
- 성능은 인간 라벨을 이용한 out-of-fold 교차검증 결과입니다.
- 쉬운 검증표본에서 과도하게 높은 임계값이 선택되는 것을 막기 위해 검토 임계값은
  최대 0.50, 저우선순위 경계는 그 절반 이하로 제한하는 안전 완충을 적용합니다.

[권장 데이터]
- 제목만으로도 실행 가능하지만 제목+초록이 더 안정적입니다.
- 최소 요건은 인간 Include/Exclude 각 10편이며, 실제 운용은 각 20편 이상을 권장합니다.
- 새로운 PICO 프로젝트는 이전 프로젝트 모델을 그대로 재사용하지 말고 새 라벨로 학습하세요.

[생성 파일]
- SR_Triage_FINAL_RESULTS.xlsx: 전체 결과와 시트별 분류
- SR_Triage_PERFORMANCE_DASHBOARD.png: PR/ROC/혼동행렬/확률분포/fold 안정성
- SR_Triage_THRESHOLD_TRADEOFF.png: Recall-NPV-검토량 절감의 임계값 관계
- SR_Triage_MODEL.joblib: 해당 프로젝트 모델
- SR_Triage_FINAL_PACKAGE.zip: 위 파일 전체 묶음
