SR Safe-Exclude Triage Platform
Version: 1.0.2
Build date: 2026-07-14

Core method:
- Word TF-IDF + balanced logistic regression
- Character TF-IDF + balanced logistic regression
- PICO/exclusion-criteria similarity model
- Weighted ensemble and group-aware out-of-fold validation
- Recall-constrained dual thresholds with deployment safety caps

Fix in 1.0.1:
- Existing input columns named "No." are preserved as "Source_No." and no
  longer cause a duplicate-column insertion error.

Change in 1.0.2:
- The initial labeling workbook is named "1차_제목_라벨링용.xlsx".
- Its first sheet contains only 순번, 연도, 제목, and 판정(o/x).

Fix in 1.0.3:
- CRITERIA OOF scoring now refits its TF-IDF vectorizer on train-only text
  each fold, matching WORD_TFIDF/CHAR_TFIDF. Previously the criteria
  vectorizer was fit once on the full labeled+unlabeled corpus, so
  validation-fold text leaked into its IDF statistics and slightly
  inflated the reported OOF recall/precision for CRITERIA and ENSEMBLE.
- matplotlib now explicitly uses the headless "Agg" backend, since figures
  are generated inside a Streamlit worker thread and GUI backends are not
  thread-safe.
- Exclusion criteria are no longer required to run the pipeline (PICO-only
  projects are valid); a warning is shown instead if both are empty.
- Selecting the same column as both title and human-label column when
  merging an external label file is now blocked instead of silently
  producing a broken merge.
